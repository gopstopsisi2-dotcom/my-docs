\## 2. Компоненты документации



| Компонент | Инструмент | Входные данные | Выходные данные |

|-----------|------------|----------------|-----------------|

| Пользовательская документация | MkDocs + Material | Markdown файлы | HTML сайт |

| Техническая документация | Doxygen | Комментарии в коде | HTML страницы |

| API документация | Swagger/OpenAPI | openapi.yaml | Swagger UI |

\## 3. CI/CD процесс



\### Триггеры



\- `push` в ветку `main` → полный пайплайн с деплоем

\- `pull\_request` в ветку `main` → проверки без деплоя



\### Этапы пайплайна



1\. \*\*Установка зависимостей\*\* (Python, Node.js)

2\. \*\*Линтинг Markdown\*\* (markdownlint)

3\. \*\*Проверка орфографии\*\* (cspell)

4\. \*\*Проверка битых ссылок\*\* (markdown-link-check)

5\. \*\*Валидация OpenAPI\*\* (swagger-cli, spectral)

6\. \*\*Сбор метрик\*\* (docstr-coverage, собственный скрипт)

7\. \*\*Сборка MkDocs\*\* (mkdocs build)

8\. \*\*Деплой на GitHub Pages\*\* (peaceiris/actions-gh-pages)



\### Артефакты



\- Собранный сайт (папка site) сохраняется как артефакт на 7 дней

\- Метрики качества сохраняются в JSON



\## 4. Стратегия версионирования



\- Используется плагин mike для мультиверсионности

\- Теги в Git: v1.0.0, v1.1.0, v2.0.0

\- Пользователь переключает версии через выпадающий список на сайте

\- LTS версии поддерживаются 24 месяца



\## 5. Стратегия локализации



\- Плагин mkdocs-static-i18n

\- Поддерживаемые языки: русский (основной), английский, китайский

\- Структура папок: docs/ru, docs/en, docs/zh

\- Скрипт extract\_i18n\_strings.py для извлечения строк перевода



\## 6. Роли и ответственность



| Роль | Ответственность | Инструменты |

|------|-----------------|-------------|

| Технический писатель | Написание документации, проверка качества | Markdown, MkDocs, AsciiDoc |

| Разработчик | Документирование кода, поддержка OpenAPI | Doxygen, JSDoc, OpenAPI |

| DevOps | Настройка CI/CD, мониторинг | GitHub Actions, Spectral |



\## 7. Качество документации



\### Метрики



\- Покрытие кода документацией: цель 90%

\- Битые ссылки: 0

\- Ошибки линтинга: 0

\- Ошибки орфографии: менее 5



\### Инструменты проверки



\- markdownlint — стиль Markdown

\- cspell — орфография

\- markdown-link-check — битые ссылки

\- spectral — валидация OpenAPI

\- docstr-coverage — покрытие кода
